# stackInterop: Cross-Chain Identity Registry

A decentralized identity registry on the Stacks blockchain that enables users to cryptographically link their Stacks accounts to their Bitcoin addresses.

## Overview

The Identity Registry serves as a bridge for reputation and identity between the Bitcoin and Stacks ecosystems. By proving ownership of a Bitcoin address, users can unlock cross-chain benefits, participate in Bitcoin-aware DAOs, and establish a unified web3 identity.

## Features

- **Decentralized Linking**: Cryptographically verify Bitcoin address ownership on-chain.
- **Privacy-First**: Minimal data stored on-chain to maintain user privacy.
- **Reputation Layer**: Foundations for cross-chain reputation systems.

## Project Structure

- `contracts/`: Clarity smart contracts (Primary logic in `stack-interop.clar`).
- `tests/`: Vitest-based test suite for contract logic.
- `settings/`: Deployment configurations.

## Getting Started

1. **Install Clarinet**: [Clarinet Installation](https://docs.hiro.so/clarinet/how-to-guides/how-to-set-up-local-development-environment)
2. **Run Tests**: `npm run test`
3. **Check Logic**: `clarinet check`
